# Absorbed Patterns

Autodebug absorbs structure from public and local debug/review skills:

- systematic debugging: no fix before root cause.
- diagnose workflows: reproduce, trace, hypothesize, test, fix, prove.
- code review skills: classify blockers and re-check after repair.
- PR resolver skills: read issues from a ledger, fix one or a scoped batch, write back the decision.
- pipeline/debug skills: output failure summary, root cause, fix, prevention, and troubleshooting.
- skill marketplace structures: keep the main skill small and put reusable contracts in references/templates.

Autodebug deliberately does not absorb check-only behavior from `autocheck`; checking can produce FAIL records, but autodebug owns the repair loop.
