# Absorbed Patterns

Autocheck absorbs structure from several public and local skill patterns:

- QA/check skills: identify the shape of the running target before choosing browser/API/CLI/data tools.
- Verification-before-completion skills: no pass claim without fresh evidence from the current run.
- Review/checklist skills: explicit severity or result states prevent vague "looks okay" output.
- Skill marketplace structures: keep the main skill small, move reusable contracts to references and templates.

Autocheck deliberately does not absorb default repair behavior from debug skills. FAIL records should be actionable, but product fixes belong to `autodebug`.
